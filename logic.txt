Функция создания нашей БД
    подключаемся к БД
        с помощью скрипта соединения выполняем код в скобках
            Создаем 2 таблицы (юзерор и реквестов)

Вызываем создание БД

get /:
    редиректим к /login
 
get logout:
    удаляем куки
    редиректим к /login

get register:
    возвращаем шаблон register.html - без ошибки
    
get login:
    возвращаем шаблон login.html - без ошибки    


post register (login, password, fio, phone, email):
    ошибка = ничего
    
    если логин не соответствует (r"[a-zA-Z0-9]{6,}", login):
        ошибка = "Логин: Латиница и цифры, 6 символов минимум"
    иначе если пароль < 8:
        ошибка = "Пароль: 8 символов минимум"
    иначе если ФИО не соответствует (r"[а-яА-ЯёЁ\s]+", fio):
        ошибка = "ФИО: Кириллица и пробелы"
    иначе если телефон не соответствует формату (r"8\(\d{3}\)\d{3}-\d{2}-\d{2}", phone):
        ошибка = "Телефон: Формата 8(XXX)XXX-XX-XX"
    иначе если email не соответствует (r"[^@]+@[^@]+\.[^@]+", email):
        ошибка = "Почта: Неверный формат"
    иначе:
        подключаемся к БД
        ищем пользователя с таким логином
        
        если пользователь не найден:
            инстертим данные в юзер(БД)
            редиректим на /login
        иначе:
            ошибка = "Логин уже занят"
    
    возвращаем страницу register.html с ошибкой (если есть) и введёнными данными


post login (login,password):
    Если логин == Админ
        Если пароль == КорокНЕТ
            сохраняем в реках что это реально тру Админ
            редиректим к /admin
        Если пароль не совпал:
            возвращаем шаблон login.html - с ошибкой что данные неверные    

    подключаемся к БД
        user = все данные из юзер где логин совпал  - fetchone()
    Если user вообще есть И пароль юзера == password (Который мы получили с фронта)
        сохраняем в куках id данного пользователя
        редиректим к /profile
    возвращаем шаблон login.html - с ошибкой что данные неверные    

get profile
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    
    
    подключаемся к БД
        в request_list (Переменная в которую мы сохраним результат запроса в БД) сохраняем все данные о курсах текущего пользователя - fetchall()
    возвращаем шаблон profile.html с данными из request_list

2 переменных - ВСЕ КУРСЫ  и ВСЕ СПОСОБЫ ОПЛАТЫ

get create_request
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    

    возвращаем шаблон create_request.html +  ВСЕ КУРСЫ иВСЕ СПОСОБЫ ОПЛАТЫ 


post create_request (course,date,payment)
    Берем id пользователя из куков
    Если его нет
        редиректим к /login    

    подключаемся к БД
        сохраняем данные в БД из формы в requests + user_id 
    
    редиректим к /profile

get admin
    Смотрим  из куков пользователь админ
    Если нет
        редиректим к /login    
    
    подключаемся к БД
        в request_list читаем все заявки, логины юзеров, их фио с заявок, + пользователи где id пользователей = заявкам к котором привязан пользователь - fetchall()
    возвращаем шаблон admin.html с данными из request_list
